            Steve Trutane • 7 months ago • edited

            Nice summary! Another place I've noticed where line continuation comes into play for improved readability is to breakup a set of chained method calls. You could jam them all together into a single line, but with multiple methods and arguments, the line can become hard to grok.

            For example, here's a series of Pandas DataFrame operations, each of which returns a modified DataFrame object which is then processed by the subsequent method call. Splitting them across multiple lines prefixed with a dot helps understand what is happening:

            dataframe.filter(cols)
            .sort_values(cols)
            .drop_duplicates(cols)

            NOTE: In the above code block, the dots should all line up vertically. Disqus apparently does NOT preserve leading whitespace in code blocks :(
